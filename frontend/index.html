Debate Protocol

5 deterministic Python rules — no LLM in any decision. Pure logic, fully auditable.

1
Physics beats vision
If a deterministic physics model (Coffin-Manson, Black, Griffith) predicts failure with P(fail) > 15%, it overrides a "looks fine" CV result with confidence <80%. Visual inspection cannot see latent damage.

if p_fail > 0.15 and cv_invoked and cv_ok is False and cv_conf < 0.80 → KILL
2
Process beats specification
If SPC shows process drift >3σ from centre, override specification compliance. A process drifting 3σ today will be out of spec tomorrow.

if process_sigma_drift > 3.0 → KILL
3
Worst-case wins for safety-critical
For AEC-Q100 (automotive), medical, and aerospace lots — take the most pessimistic estimate from any tool. 10 DPPM = 0.001% threshold.

if app in {"automotive","medical","aerospace"} and max(p_fails) > 0.00001 → KILL
4
Weighted average for non-safety-critical
For consumer and industrial lots, blend all tool outputs. If blended result exceeds the application threshold, flag (not kill).

consumer threshold 0.1% · industrial 0.005% → FLAG if exceeded
5
Tie-breaker: Orchestrator LLM
If no conflict detected (rules 1–4 did not fire), clean pass. The LLM writes the explanation; the rules made the decision. LLM output is advisory only — the final_decision field is hard-enforced by the Python code after the LLM responds.

Architecture

Person 3's subsystem — how data flows from lot input to final report.

File Structure
packguard/ ├── main.py ← FastAPI app (POST /orchestrate, /debate, /aggregate) ├── lot_schema.py ← Pydantic LotState model — shared API contract with Person 2 ├── debate.py ← 5 deterministic debate rules (no LLM) ├── aggregator.py ← 1−∏(1−Pi) aggregation + DPPM thresholds ├── orchestrator/ │ ├── service.py ← run_orchestrator() — main brain function │ └── prompt.py ← SYSTEM_PROMPT constant (writer instructions) ├── kb/ │ ├── cases.json ← 35 curated failure cases (JEDEC-sourced) │ ├── embed.py ← Build FAISS index: python kb/embed.py │ ├── retriever.py ← retrieve(query, k) → top-k similar cases │ ├── index.faiss ← Generated by embed.py │ └── cases_meta.pkl ← Generated by embed.py ├── pdf_gen/ │ └── generator.py ← generate_pdf(report, lot_id) → PDF path └── tests/ └── test_all.py ← 24 unit tests — all passing
Orchestration Flow
Person 2 pipeline output (LotState JSON)
  ↓
POST /orchestrate
  ↓
[1] run_debate(lot) → DebateResolution (deterministic, <5ms)
  ↓
[2] aggregate_failure_probability() → overall P(fail) (deterministic)
  ↓
[3] retrieve_for_lot() → top-3 KB cases (FAISS, ~50ms)
  ↓
[4] _call_anthropic(context) → narrative JSON (LLM, ~2s, temperature=0)
  ↓
[5] Hard-enforce debate override on LLM output (deterministic)
  ↓
Structured report JSON → Person 4's dashboard + PDF
